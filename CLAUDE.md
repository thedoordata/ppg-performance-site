# PPG Performance — Cinematic Landing Page

Лендинг для PPG Performance — тюнінг-ательє та СТО преміум-класу в Києві (Stage 1–4, чіп-тюнінг, діагностика). Аудиторія — власники Porsche/BMW/Mercedes. Повне творче ТЗ лежить у `docs/TZ.md` — звіряйся з ним для деталей по кожній сцені, тону тексту й переліку послуг. Цей файл — лише практичний орієнтир для збірки.

## Стек

- Next.js (App Router, static export де можливо) + TypeScript — важлива SEO-індексація (локальний автосервіс шукають в Google)
- Tailwind CSS
- GSAP + ScrollTrigger — покадрова анімація, прив'язана до прогресу скролу
- Lenis — плавний (inertia) скрол
- Шрифти через next/font

## Сцени (коротко; повна розкадровка — docs/TZ.md розділ 4)

0. Hero — закриті ворота, жовта окантовка, лого
1. Ворота відкриваються (перехід)
2. Двигун / ТО — капот відкрито
3. Підвіска / рульове — авто на підйомнику
4. Гальма — колесо крупним планом
5. Трансмісія / приводи
6. Вихлоп / турбо / Stage-тюнінг — ключова "м'ясна" сцена, дим/спалахи
7. Оптика — фари довертаються до камери
8. Екстер'єр — 360°, обвіс, диски, тонування
9. Кабіна / мультимедіа
10. Фінал — три авто в ряд, CTA, контакти

## Типографіка

- Display (заголовки сцен, капс): Druk Wide / Neue Machina — якщо немає ліцензії, підібрати близький безкоштовний аналог (напр. Anton, Bebas Neue Wide) і позначити як тимчасову заміну
- Body: Inter / Suisse Int'l
- Utility/дані (тахометр-скролбар, номери сцен): JetBrains Mono

## Порядок реалізації (робити фазами, не все одразу — після кожної фази зупинись і покажи результат)

1. Скаффолд Next.js + TypeScript + Tailwind, підключити шрифти й палітру як CSS-змінні
2. Hero-сцена (статична) + фіксований header
3. Механіка скролу: Lenis + GSAP ScrollTrigger, пінінг секцій, синхронізація «відкриття воріт» з прогресом скролу 0–100%
4. Заготовки сцен 2–9 як послідовні pinned-секції з плейсхолдерами замість реальних фото/відео (суцільний колір + підпис «сцена N: що тут має бути»)
5. Фірмовий елемент — тахометр-скролбар (див. ТЗ п.5.3): голка обертається пропорційно прогресу сторінки
6. Фінальна сцена + блок контактів/CTA (адреса, телефон, Instagram, графік роботи — з ТЗ)
7. Адаптив під мобільні — спрощена анімація, статичні fallback-кадри замість відеопослідовностей
8. Полірування: letterbox-переходи між макро-сценами, прелоадер у стилі «запуску двигуна», кастомний курсор (опційно)

## Асети

Реальних фото/відео поки немає — використовувати плейсхолдери до отримання матеріалів від клієнта. Структура: `/public/scenes/00-hero/`, `/public/scenes/01-open/`, `/public/scenes/02-engine/` … `/public/scenes/10-final/`.

## Важливі обмеження

- Жодного autoplay відео зі звуком без дії користувача
- Анімація деградує на мобільних, а не вимикається повністю
- Уникати типових SaaS-прийомів (картки зі скругленням+тінню+іконкою) — сайт має відчуватись як кінотрейлер, а не корпоративний лендинг
- Продуктивність: важкі відеосцени не повинні блокувати LCP; лінива підвантажка секцій поза viewport
